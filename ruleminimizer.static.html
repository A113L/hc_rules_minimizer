Phase 2: Levenshtein Redundancy Filtering (Optional: --levenshtein-max-dist)

Reduces "semantic" duplicates by removing rules that are too similar to a more frequent rule based on Levenshtein distance. This is applied before functional minimization.

Sorted Unique Rules (Textual)
↓
Levenshtein Filter (Max Dist)
↓
Semantically Unique Rules

Levenshtein Filter Impact

Shows the reduction in rules after applying the Levenshtein filter.

Phase 3: Rule Filtering & Minimization Modes

The core of the script offers multiple filtering modes, including a powerful functional minimization using a Hashcat RuleEngine simulator.

Semantically Unique Rules
↓
Mode 1: Min Occurrence
Mode 2: Max Rules (Top N)
Mode 4: Inverse (Bottom N)
↓
Mode 3: Functional Minimization
(Hashcat RuleEngine Sim + Multiprocessing)
↓
Functionally Unique Rules

Functional Minimization & Pareto Analysis

Functional minimization reduces rules based on their actual effect on a test vector. Pareto analysis helps identify optimal cutoff points.
